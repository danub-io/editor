⚡ Fix O(N*M) lookup in RightSidebar and Kanban components

* Pre-computed the `planningSections` mapping as a dictionary using `useMemo` in `RightSidebar`.
* Refactored lookups for sections to be O(1) inside looping render structures to prevent O(N*M) slowdowns.
* Fixed typing issue where implicit `any` parameter 'r' caused strict ts build errors.
* Re-ran render benchmark confirming O(N) optimizations resulted in rendering latency being halved for RightSidebar components with numerous sub-items.
